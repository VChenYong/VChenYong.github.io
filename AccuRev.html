1 ACCUREV NOTE AND TRICK

1.1 command line update workspace

  • call accurev update

1.2 revert to back

  • if only revert back one file among other files in a transaction, then it is better undo the modification and promote it again.
  • if we need to revert back all files from one transaction , then we can do revert back form the the actual stream . after all the files are been reverted , the transaction will be disappeared.

1.3 revert one transaction in workspace only

  1. we can re-parent workspace or create a new workspace under the stream where the transaction can be found.
  2. show all history transaction
  3. right click on the transaction we want to revert
  4. select revert

1.4 reset password

# In cmd mode run the following command
  Accurev login -n
# Then you’ll be prompted for username//password.

1.5 cross promoting

  • select your file drag-and drop to the stream level you want (or change Palette)
  • If the Promote command is enabled when you select all the elements, you can proceed to propagate all the issues (and their versions) to the destination stream
  • detail see the manual pdf

1.6 Move WorkSPaces

Ok, So I deleted the workspace in the stream (in AccuRev client)… or so I thought. AccuRev does not actually remove the workspace; AccuRev deactivates the workspace.

So first I had to reactivate it (using CLI): 

accurev reactivate wspace foo_workspace

accurev reactivate wspace fooworkspace Then I had to change the location of the workspace in accurev 

C:\EclipseWorkspace>accurev chws -w foo_workspace -l

Notes: -w is the name of your workspace (must be yours, can't be someone else's; there is -s for that) -l is the location you want to change the physical location too #+ENDEXAMPLE

1.7 useful commands

1.7.1 Login to accurev

accurev login [ -n ] '<user-name>' '<password>'

1.7.2 Login with another server name

accurev login -H '<servername>:<port>' '<username>' '<password>' The –n option creates a non-expiring session

1.7.3 Create stream

accurev mkstream -s <stream> -b <backing-stream>

e.g.:

accurev mkstream -s dev.user.MyStream -b dev.user.MyBasisStream

1.7.4 Create workspace

accurev mkws -w <workspace-name> -b <backing-stream> -l <location>

e.g.:

accurev mkws -w dev.ws.user.MyWorkspace -b dev.user.MyStream -l /home/user/ar/dev.ws.user.MyWorkspace

1.7.5 Update workspace

cd /home/user/ar/dev.ws.user.MyWorkspace

accurev update

1.7.6 Change workspace

accurev chws -w <workspace> <new-name>

accurev chws -w <workspace> -l <new-location> -m <new-machine>

1.7.7 Information about current location

accurev info

1.7.8 Show command

show [ -f <format>] [ -p <depot-name>] [ -d ] [ -s <stream> [ -R ] ] [ -t <time-spec>] streams

show [ -f <format>] [ -a ] wspaces

1.7.9 View information about files in workspace

1.7.10 Show overlaps:

accurev stat -o -B

1.7.11 Show missing files:

accurev stat -M

1.7.12 Forcedly update accurev workspace when local files were missed:

accurev pop -O -R .

1.7.13 Upgrade accurev client to latest version

accurev upgradeclient -s
