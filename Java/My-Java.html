    • Serializable 101 

      Java Object Serialization, introduced as part of the ground-breaking feature set that made up JDK 1.1,
serves as a mechanism to transform a graph of Java objects into an array of bytes for storage or transmission,
such that said array of bytes can be later transformed back into a graph of Java objects.

In essence, the idea of Serialization is to "freeze" the object graph, move the graph around (to disk,
across a network, whatever), and then "thaw" the graph back out again into usable Java objects.
All this happens more or less magically, thanks to the ObjectInputStream/ObjectOutputStream classes,
full-fidelity metadata, and the willingness of programmers to "opt in" to this process by tagging their
classes with the Serializable marker interface.
