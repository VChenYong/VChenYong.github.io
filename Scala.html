1.1 Scala Functions

1.1.1 definition

  • A function is a mapping that takes one or more arguments and produces a single result.
  • exmaple 
    (x:Int, y:Int) => x+y

// another examplePythagorean triple

 aSquare+bSquare = cSquare
}
  • A function is defined by using an equation that gives a name for the function, a name for each of its arguments , and a body that specifies how the result can ve calculated in term of arguements. 
    val power :(Int,Int)=>Int =

1.1.2 ways to define scala functions

  1. ways to define functions
    1. val + function Name : ( List of argument Types) => return Type = (list of arguments)=> function body 
      val add: (Int,Int)=>Int = (x,y)=>x+y
    2. val +function name :(list of Arguments and its types)=> function body 
      val add = (x:Int,y:Int)=>x+y
    3. Use function trait 
      val add:Function3[Int,Int,Int] = (a:Int,b:Int)=> a+b
//add: (Int, Int) => Int = <function2>
    4. Implementing the appropriate function trait and define it's required Apply method. 
      val add: (Int,Int)=>Int = new Function2 [Int,Int,Int]{
    def apply(a:Int,b:Int):Int={a+b}
}
    5. Create a new class which extends a certain function 
       class add (a:Int,b:Int) extends Function2[Int,Int,Int]{
   def apply(a:Int,b:Int):Int={a+b}
}
  2. ways to define functions
    • def + function Name : ( List of argument Types) => return Type = (list of arguments)=> function body 
      def add: (Int,Int)=>Int = (x,y)=>x+y
//add: (Int, Int) => Int
    • def +function name :(list of Arguments and its types)=> function body 
      def add = (x:Int,y:Int)=>x+y

1.2 Scala closures

1.2.1 Closure

  1. A function contains at least one free varaible is called an open term. In order to get to a fullly functional function, all free varibals have to be bound( turning that open term into a closed term). For this to be done, the compilier reaches out to the so called lexical envirnment , in which that function was defined and tries to find a binding. This process is called closing over which results in a closed expression or for short Closure
  2. Closure is Scala is dynamic blinding. which means free varibale bind to relative locations in memory that can store values. Attention: this can create side effect. example: 
    var minAge = 18
val isAdult = (age:Int)=>age>=minAge
val isQuebecAdult = isAdult(20) // =>true
minAge = 21
val isCanadianAdult =isAdult(20) // =>false

1.5.1 Function Literals1

  • Function literals are anonymous – they don't have a name by default, but you can give them a name by binding them to a variable.
  • A Function is a mapping that takes one or more arguments and produces a single result (typically by a function definition that specify how the result can by calculated solely in terms of its arguments).
  • example of function literal 
(  x :Int,  y :Int  )  =>  x  +  y

1.5.2 Functions as first class values

  • a Function is only allowed to depend on the given parameters
  • a method may refer to the internal state of the object it’s a member of or – even worse – may alter the internal state of that object. which can be seen as an evil side effect
  • every function has a type
  • a method is only a member of a certain object. The object itself is of course an instance of a certain type, but the method itself has no type
  • Since function has a type like any other types, so you can pass function as parameters to a function or as a result of a function.

  • function canbe seen as a discrete value in its own right

  • (Sub-)Type polymorphism and Parametric polymorphism (you may call it Generics). 
//underscore used to designate a so called Existential Type: we’re informing the compiler, that there must exist such a concrete type, but //we don’t care which type it is – it might be a List of Strings or a List of Booleans, we simply don’t care.
val empty = ( xs :List[_] ) => xs == Nil



val names : List[String] = List( "Anne", "George", "Carla" )
val primes : List[Int] = List( 11, 17, 19 )

List(empty( Nil ),           // true
